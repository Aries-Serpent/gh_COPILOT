# Communication Standards

This project expects consistent logging and documentation conventions.

## Logging
- Use functions from `utils.log_utils` for console output.
- Prefix messages with a timestamp generated by `_log_plain`.
- Select `level=logging.INFO`, `logging.WARNING`, or `logging.ERROR` as appropriate.

## Documentation
- Provide concise docstrings for complex functions, listing parameters and expected return values.
- Avoid ambiguous language in README or comments; use imperative instructions.

## Code Style
- Follow existing snake_case function names and CamelCase classes.
- Keep line length under 120 characters.
